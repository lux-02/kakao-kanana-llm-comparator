# Kanana LLM Comparator

Kanana와 다른 주요 LLM을 한 화면에서 비교하고, 간단한 노드 기반 워크플로우까지 실험할 수 있는 웹앱입니다.

- `비교` 페이지: 같은 프롬프트를 여러 모델에 동시에 보내고 결과를 나란히 비교
- `워크플로우` 페이지: 노드를 추가하면서 단계별 LLM 흐름 실험
- 지원 프로바이더: `Kanana`, `OpenAI`, `Google Gemini`, `Anthropic Claude`

이 프로젝트는 **각 사용자가 자신의 API 키를 직접 입력해서 사용하는 BYOK(Bring Your Own Key)** 방식으로 동작합니다.

## 왜 만들었나요?

Kanana는 한국어 톤과 표현을 다듬는 데 강점이 있고, GPT / Gemini / Claude는 구조 생성과 추론에서 각각 다른 성격을 보입니다.  
이 차이를 더 직관적으로 비교하고, 개발 경험이 많지 않은 사람도 쉽게 실험할 수 있도록 만든 도구입니다.

## 주요 기능

### 1. 모델 비교

- Kanana 고정 카드 + 원하는 비교 모델 추가
- 스트리밍 응답 지원
- 프로바이더별 API 키 입력
- 세션 동안 API 키 유지
- 모바일 대응

### 2. 워크플로우

- 시작 노드에서 프롬프트 입력 후 전체 실행
- 노드 위/아래 추가
- 분기 최대 2개 지원
- Merge 노드로 두 브랜치 결과 결합
- 노드별 모델 선택 / 시스템 프롬프트 설정

## 지원 모델

현재 앱에는 아래 계열이 연결되어 있습니다.

- `Kanana`
- `GPT-5.4`, `GPT-5.4 Mini`, `GPT-5.4 Nano`, `GPT-4.1`
- `Gemini 2.5 / 3.x` 계열 일부
- `Claude Opus / Sonnet / Haiku` 계열 일부

실제 노출 모델 목록은 앱 내부 드롭다운을 기준으로 확인하는 것이 가장 정확합니다.

## 사용 방법

### 비교 페이지

1. Kanana API 키를 입력합니다.
2. 프롬프트를 입력합니다.
3. `+` 버튼으로 비교할 모델을 추가합니다.
4. 필요한 프로바이더의 API 키를 입력합니다.
5. `비교`를 눌러 결과를 확인합니다.

### 워크플로우 페이지

1. 시작 노드에 프롬프트를 입력합니다.
2. 아래로 노드를 추가하거나 분기를 만듭니다.
3. 각 노드에서 모델과 시스템 프롬프트를 설정합니다.
4. 필요한 API 키를 입력합니다.
5. `전체 실행`으로 흐름을 실행합니다.

## API 키와 보안

이 프로젝트는 사용성을 높이되, 키를 서버에 저장하지 않는 방향으로 설계했습니다.

- API 키는 **서버 DB에 저장하지 않습니다**
- API 키는 **브라우저 세션(sessionStorage)** 에만 유지됩니다
- 새로고침해도 같은 세션에서는 유지됩니다
- 브라우저 세션이 종료되면 키는 사라집니다

그래도 중요한 점이 있습니다.

- 이 앱은 실제 프로바이더 호출을 수행하는 웹앱입니다
- 민감한 API 키를 다룰 때는 공개 배포 환경보다 **로컬 실행**을 더 권장합니다
- 비용이 발생하는 키는 반드시 본인 책임 하에 사용해야 합니다

## 배포

이 프로젝트는 Next.js 기반이라 Vercel에 바로 배포하기 좋습니다.

다만 이 앱은 사용자 API 키를 직접 입력받는 구조이므로, 공개 배포 시에는 아래 내용을 함께 안내하는 것을 권장합니다.

- 서버에 키를 저장하지 않음
- 브라우저 세션에만 유지됨
- 개인 키 사용에 따른 비용은 사용자 본인 부담

## 프로토타입 안내

이 프로젝트는 현재 프로토타입 단계입니다.

- 비교 기능은 실사용 가능한 수준으로 동작하지만, 모델별 응답 성격 차이는 그대로 드러납니다
- 워크플로우 기능은 데모와 실험에 초점을 둔 v1입니다
- 긴 회의록이나 장문 입력에서는 일부 모델이 최대 출력 토큰 제한으로 인해 응답이 중간에 종료될 수 있습니다
- 현재는 자동 이어쓰기(continuation)가 구현되어 있지 않습니다

긴 입력을 다룰 때는 아래 방법을 권장합니다.

- 입력 원문을 조금 더 짧게 정리해서 넣기
- 고급 설정에서 `max tokens` 값을 높이기
- 긴 결과물을 한 번에 만들기보다 단계별로 나누어 워크플로우 구성하기

## 주의 사항

- 프로바이더별 응답 속도와 스트리밍 방식은 다를 수 있습니다
- 일부 preview 모델은 정책이나 availability가 바뀔 수 있습니다
- 사용 중인 모델 이름과 API 정책은 시간이 지나면 달라질 수 있습니다
- 사용자가 입력한 API 키로 발생한 비용은 사용자 본인 책임이며, 이 프로젝트는 해당 비용을 책임지지 않습니다
